# loop engine 全役に共通する原則

このファイルは `loop-roles` プラグインの各 skill から `@${CLAUDE_PLUGIN_ROOT}/shared/principles.md` で参照される共通指針。役固有の指針は各 SKILL.md 側に書く。

## 呼び出しの前提

- このスキルは **loop engine の runner からの明示呼び出し**(`/loop-roles:<name>` slash)でのみ起動する。`disable-model-invocation: true` でモデル自動発火は禁じてある。
- 動的文脈(タスク契約・diff・規範・過去 run ブリーフ等)は slash 呼び出しの引数(SKILL.md 末尾の入力ブロックに展開される)で渡される。役ごとの構造は各 SKILL.md の冒頭で定義する。

## 出力・言語

- **報告は日本語**。コード内コメントを書くなら日本語(global の CLAUDE.md 規約に従う)。
- **コメントは WHY のみ**。「何をしたか」のコメントは残さない(コード読めば分かる)。差分の事後説明やプロセス報告のコメントを残さない。
- 構造化出力(`--json-schema`)が指定されている役は **スキーマに必ず従う**(欠落フィールドを作らない、enum 外の値を返さない)。

## ファイル / パス / 実行系

- 作業は **隔離された git worktree 内**で行われる(serial repo モードのときは repo 本体)。
- リポジトリ内ファイルを指すパスは **リポジトリルートからの相対パス**で書く。**絶対パス禁止**(`/Users/...` や `~/` を直書きしない)。worktree ではなく別の場所を指してしまう。
- 各役のツール制約に従う(read-only 役は Read/Grep/Glob のみ。変更系ツールは禁止)。

## 文脈の優先順位

注入される文脈の信用順位は固定:

```
CLAUDE.md(リポジトリ憲法) > conventions.md(承認済み規範) > 過去 run の事実ブリーフ
```

引数の入力ブロックに上記の各セクションが含まれることがある。CLAUDE.md は Claude Code がネイティブに拾うので明示的には注入されない場合がある。

## 安全側に倒す不変条件

- **自己申告を信じない / 死角を作らない**: 自動で pass/fail と言い切れないときは安全側(`handoff` / `revise`)に倒す。「だいたい OK」「動くはず」で `pass` にしない。
- **test gaming を疑う**: テストを通すためだけにテストを書き換える、本質を解かずに楽な側へ倒す、空通りテストで茶を濁す等は **検出して revise / fail / handoff の対象**。
- **検証可能性が最優先**: 数値・コマンドで二値判定できる証跡を残す。曖昧な検証で完了報告しない。
